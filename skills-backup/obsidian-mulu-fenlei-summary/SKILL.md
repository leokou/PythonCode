# Obsidian 目录分类与文件摘要管理

目录分类 + File Summary 双引擎：整理目录文件分类 → 生成/维护 ✍️ 文件索引摘要。

**独立工具**：用户可单独调用处理任何目录文件夹。不在 Pipeline 主流程中。

**例外**：Pipeline 检测到"标题不变+内容变化"时，会触发本 Skill 重生成 File Summary（轻量路径）。

上游：无（独立工具） / `obsidian-pipeline`（轻量路径触发时）
下游：无（最终输出）

## 触发词去歧义（重要）

| 触发词 | 触发的 Skill | 说明 |
|--------|--------------|------|
| "摘要目录 xxx" / "目录摘要" / "分类目录 xxx" / "文件摘要" | **本 Skill** (Mulu-fenlei-summary) | 生成 ✍️ File Summary + 目录分类 |
| "整理 xxx 目录" / "整理 AI 目录" / "整理知识" | **Organizer** | 归属目录 + 移动文件 + 建立双链 |
| "重命名 xxx" / "批量重命名" | **Fire-rename** | 规范化文件名 + 双链维护 |
| "处理 Capture" / "运行流水线" | **Pipeline** | 调度器，调用 Planner → Compiler → Organizer |

**禁止使用模糊触发词**："整理目录文件"（已弃用，容易与 Organizer 混淆），改用"摘要目录 xxx"。

## 双层架构

| 层级 | 文件 | 用途 | 写入者 |
|------|------|------|--------|
| 人类可读层 | `📖目录 索引.md` + `🧩 目录-xxx.md` | 浏览导航、File Summary 展示 | Mulu-fenlei-summary（直接写入 ✍️ 摘要） |
| 机器可读层 | `logs/summary/summary-state.json` | 记录目录状态、文件数、摘要数 | Mulu-fenlei-summary |

**读取优先级**：
1. 先读索引（快速判断哪些已有摘要）
2. 再读状态文件（精确检测目录变化）

## ⚠️ 强制执行清单（必须全部完成）

**每次执行本 Skill，必须按顺序完成以下 4 步，缺一不可。**

### Step 1：文件内部摘要（源文件）

| 项目 | 说明 |
|------|------|
| **操作对象** | 目录中的每个 `.md` 源文件 |
| **操作位置** | 文件第二行（标题 `# xxx` 之后） |
| **格式** | `>✍️ 一句话总结` |
| **检查机制** | 不存在 → 读取正文生成摘要 → 写入；已存在 → 跳过 |

### Step 2：目录索引摘要（目录文件）

| 项目 | 说明 |
|------|------|
| **操作对象** | `🧩 目录-xxx.md` 文件 |
| **操作位置** | 每个 `[[链接]]` 之后 |
| **格式** | `[[文件名]] ✍️ 一句话总结` |
| **检查机制** | 链接后无 ✍️ → 补充；有 ✍️ → 保留 |

### Step 3：索引更新（同步写入 📖目录 索引.md）

| 项目 | 说明 |
|------|------|
| **操作对象** | `📖目录 索引.md` 中对应分类区域 |
| **操作位置** | 找到对应分类 → 追加新条目 / 更新已存在条目 |
| **格式** | `[[文件名]] ✍️ 一句话总结 \| 更新:YYYY-MM-DD \| 状态:已整理` |
| **检查机制** | 文件不存在 → 追加；已存在无 ✍️ → 更新；已有 ✍️ → 跳过（保护规则） |

详细写入规则见下方"索引写入规则"章节。

### Step 4：完成验证（必须报告）

**执行完毕后，必须输出以下验证报告：**


## 工作流程

```
Step 1: 遍历目录中每个源文件 → 检查第二行是否有 >✍️ → 无则读取正文生成摘要 → 写入第二行
Step 2: 更新目录文件 → 每个[[链接]]后补充 ✍️ 摘要
Step 3: 更新索引 → 将整理结果写入 📖目录 索引.md（✍️ + 更新时间 + 状态）
Step 4: 输出执行报告 → 列出扫描数、新增数、遗漏数、索引更新数
```

## File Summary 规范

### 定义

File Summary = 文件索引摘要，用于人和 AI 快速理解文件价值。

**不是**：文件备注、简短描述、标签、标题重复
**是**：一句话总结 + 核心要点 + 关联文件

### 生成公式

```
✍️ 一句话总结（必填）
- 核心要点1
- 核心要点2
- 核心要点3
关联：[[相关文件1]] [[相关文件2]]
```

### 示例

| 文件名 | File Summary |
|--------|--------------|
| 再审申请书 - 定稿 @ 上级法院正式版 | ✍️ 上级法院提交版本的再审申请文件 |
| | - 用于说明案件再审请求和法律依据 |
| | - 包含事实认定、程序问题及申请理由 |
| | 关联：[[民事起诉状 - 初稿]] [[证据清单]] |
| Docker - 工具 @ 容器部署方法 | ✍️ Docker 容器化部署教程 |
| | - 用于指导从零搭建 Docker 环境 |
| | - 包含安装步骤、常用命令和 Compose 配置 |
| | 关联：[[Docker Compose - 配置]] [[Linux - 服务器]] |

### 质量标准

| 检查项 | 要求 |
|--------|------|
| 一句话总结 | ✅ 必填，说明文件是什么 |
| 核心要点 | ✅ 3-5个，说明文件包含什么 |
| 关联文件 | ✅ 可选，关联其他知识页面 |
| 是否超过一句空话 | ✅ |
| 是否方便 AI 判断 | ✅ |

不合格示例：`✍️ 资料`（无检索价值，拒绝）

## 文件内部摘要格式

所有正式 Markdown 文件，标题后第二行必须存在：

```markdown
# 文件标题

>✍️ 文件索引摘要

---

正文内容
```

### 检查机制

| 情况 | 处理 |
|------|------|
| 不存在 ✍️ | 读取正文 → 生成摘要 → 写入第二行 |
| 已存在 ✍️ | 跳过，不覆盖 |

---

## 目录索引摘要格式

目录文件中每个条目必须包含 ✍️：

```markdown
## 📁 分类名称
- [[文件名称]] ✍️ 文件索引摘要
- [[另一个文件]] ✍️ 文件索引摘要
```

### 检查机制

| 情况 | 处理 |
|------|------|
| 链接后无 ✍️ | 读取文件 → 生成摘要 → 补充 |
| 链接后有 ✍️ | 保留，不覆盖 |

## 摘要保护规则

**已存在的 File Summary 默认不可覆盖。**

优先级：

```
人工摘要 > AI生成摘要 > 自动推测摘要
```

禁止 AI 每次运行重新改写旧摘要，避免产生漂移。

## 目录分类规则

### 格式规则

```markdown
# 🧩 目录-xxx

---
title: 🧩 目录-xxx
类型: 目录
标签: [目录]
---

## 📌 一句话总结
{一句话说明这个目录包含什么内容}

## 📁 {分类1名称}
- [[文件1]] ✍️ 摘要内容
- [[文件2]] ✍️ 摘要内容

## 📁 {分类2名称}
- [[文件3]] ✍️ 摘要内容

[[🏠 home-xxx]]
```

### 分类维度优先级

1. **流程阶段**（项目/开发有明确流程时）
2. **时间轴**（有时间顺序时）
3. **重要程度**（通用场景）
4. **内容类型/属性**（辅助分类）

### 分类维度参考

| 目录类型 | 推荐分类维度 |
|----------|--------------|
| 项目类（成立业委会） | 流程阶段：前期准备 → 进行中 → 完成 |
| 工具类（AI工具） | 内容类型：教程 → 参考 → 经验 |
| 知识类（AI学习） | 知识领域：基础 → 进阶 → 实战 |
| 个人类（leo） | 属性：账号 → 工作 → 阅读 → 成长 |

## 状态追踪机制

### 追踪文件位置

```
logs/summary/summary-state.json
```

### 追踪数据结构

```json
{
  "版本": "1.0",
  "上次运行": "ISO 8601 时间戳",
  "目录状态": {
    "目录文件路径": {
      "路径": "🧩 目录-xxx.md",
      "标题": "目录文件的第一个 # 标题",
      "文件数": "当前文件数",
      "处理状态": "已更新 | 已跳过",
      "处理时间": "最后处理时间",
      "分类数": "分类数量",
      "摘要数": "摘要数量"
    }
  }
}
```

### 变化检测规则

**核心逻辑：标题变了 = 目录结构变了 = 必须重新整理**

| 优先级 | 条件 | 判断 | 动作 |
|--------|------|------|------|
| 1 | 目录文件不存在 | 新建 | 创建目录文件 |
| 2 | 目录文件的 # 标题与记录不同 | 修改（结构变了） | 重新整理 |
| 3 | 目录文件中的文件数变化 | 有增减 | 更新分类 |
| 4 | 以上全都不满足 | 未变化 | 跳过 |

**说明**：
- 目录文件内容修改但标题不变 → 跳过
- 标题修改 → 处理（目录结构变了）
- 文件数变化 → 处理（有新增或删除的文件）

### 执行后必须

1. 更新 `logs/summary/summary-state.json`
2. 写入运行日志 `logs/summary/summary-YYYY-MM-DD.md`
3. **更新 `📖目录 索引.md` 中对应条目的状态和 ✍️**

## 索引写入规则

整理目录文件后，**必须同步更新** `📖目录 索引.md`：

### 写入格式

```markdown
## 📁 分类名称
- [[文件名]] ✍️ 一句话总结 | 更新:YYYY-MM-DD | 状态:已整理
```

### 写入位置

在 `📖目录 索引.md` 中找到对应的分类区域，追加或更新条目。

### 状态值

| 状态 | 说明 |
|------|------|
| 已整理 | 文件已有 ✍️ 摘要 |
| 未整理 | 文件尚无 ✍️ 摘要 |
| 已移动 | 文件已被 Organizer 移动到正式目录 |

### 更新规则

| 情况 | 处理 |
|------|------|
| 文件在索引中不存在 | 追加新行 |
| 文件在索引中已存在但无 ✍️ | 更新行，添加 ✍️ |
| 文件在索引中已有 ✍️ | 跳过（保护规则） |

### 索引格式规范

```markdown
## 📁 {分类名称}
- [[文件名]] ✍️ 一句话总结 | 更新:YYYY-MM-DD | 状态:已整理
- [[另一个文件]] | 更新:待处理 | 状态:未整理
```

**字段说明**：

| 字段 | 说明 | 必填 |
|------|------|------|
| `[[文件名]]` | 双链 | ✅ |
| `✍️ 一句话总结` | 文件摘要 | ✅ |
| `更新:YYYY-MM-DD` | 最后更新时间 | ✅ |
| `状态:已整理/未整理/已移动` | 处理状态 | ✅ |

## 注意事项

- 编码必须为 UTF-8
- **允许修改源文件**：在标题后第二行插入 `>✍️` 摘要（这是 Step 1 的必须操作，不是"修改正文"）
- **禁止修改源文件正文**：除了第二行的 `>✍️` 摘要外，不改动源文件的其他内容
- 分类名称要简洁直观，符合"用户找文件时的第一反应"
- 如果某个文件难以归类，放入"其他"分类
- 每次操作完成后记录到日志
- **必须更新索引**：整理目录后，同步更新 `📖目录 索引.md` 中对应条目

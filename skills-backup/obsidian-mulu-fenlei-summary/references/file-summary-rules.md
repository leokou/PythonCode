# File Summary 详细规则

## 1. 定义

File Summary（文件索引摘要）是用于目录展示和 AI 理解文件价值的一句话说明。

**作用**：
- 目录导航层：让人快速了解文件价值
- AI检索层：让 AI 无需读取正文即可判断相关性

---

## 2. 标准格式

### 公式

```
✍️ 这是【什么文件】，用于【什么目的】，包含【哪些重点】
```

### 示例

| 文件名 | File Summary |
|--------|--------------|
| 再审申请书 - 定稿 @ 上级法院正式版 | ✍️ 上级法院提交版本的再审申请文件，用于说明案件再审请求和法律依据，包含事实认定、程序问题及申请理由。 |
| 物业沟通 - 成立业委会 @ 我方与物业 | ✍️ 记录业委会筹备阶段与物业沟通过程，用于跟踪双方协商事项，包含合作方案、问题反馈和处理结果。 |
| Docker - 教程 @ 容器部署方法 | ✍️ Docker 容器化部署教程，用于指导从零搭建 Docker 环境，包含安装步骤、常用命令和 Compose 配置。 |
| Claude - 教程 @ API配置方法 | ✍️ Claude API 配置教程，用于指导申请 API Key 并完成初次调用，包含认证流程、参数设置和常见错误处理。 |

---

## 3. 质量标准

### 必须包含

| 检查项 | 要求 |
|--------|------|
| 文件是什么 | ✅ |
| 文件用途 | ✅ |
| 核心内容 | ✅ |
| 区别于其他文件 | ✅ |

### 不合格示例

```
✍️ 资料
```

原因：无检索价值，拒绝。

```
✍️ 相关文档
```

原因：无法区分文件价值，拒绝。

---

## 4. 生成流程

```
读取文件
    ↓
分析：
  - 标题
  - Frontmatter
  - 章节结构
  - 正文内容
    ↓
提取核心信息
    ↓
按公式生成摘要
    ↓
质量检查
    ↓
写入文件第二行 + 目录索引
```

---

## 5. 保护规则

**已存在的 File Summary 默认不可覆盖。**

| 情况 | 处理 |
|------|------|
| 没有摘要 | 生成 |
| AI生成摘要 | 允许优化 |
| 人工修改摘要 | 保护 |
| 人工确认摘要 | 禁止覆盖 |

优先级：人工摘要 > AI生成摘要 > 自动推测摘要

---

## 6. 文件内部格式

```markdown
# 文件标题

>✍️ 文件索引摘要

---

正文内容
```

---

## 7. 目录索引格式

```markdown
## 📁 分类名称
- [[文件名称]] ✍️ 文件索引摘要
- [[另一个文件]] ✍️ 文件索引摘要
```

---

## 8. 人机协同

**人负责**：判断价值、修改摘要、最终确认

**AI负责**：初步分析、自动生成、检查缺失、维护一致性

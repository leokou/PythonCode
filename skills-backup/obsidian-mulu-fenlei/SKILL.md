# 目录分类整理 (obsidian-mulu-fenlei) - 旧版本

将 `🧩 目录-xxx` 文件中的文件列表按照分类规则重新组织、缩进展示。

上游：`obsidian-knowledge-organizer`（移动文件后自动触发）
下游：无（最终输出）

## 执行工具

- Read、Write、Edit、Glob、Grep

## 使用场景

| 场景 | 触发方式 |
|------|----------|
| 用户手动要求整理目录 | 用户说"整理目录xxx" |
| Organizer 移动文件后 | 自动触发，传入文件列表 |
| 新建目录文件后 | 用户要求分类文件列表 |

## 输入方式

### 方式1：指定目录文件（默认）

用户提供目录文件名，如"整理目录-成立业委会"。

### 方式2：接收文件列表（Organizer 调用）

从 Organizer 接收参数：

```yaml
调用参数:
  目录文件: "🧩 目录-成立业委会.md"
  新增文件:
    - "业委会 - 方案 @ 分层联络工作机制.md"
  操作类型: add  # add=新增文件, rebuild=完全重写
```

## 执行流程

1. **读取目录文件**：打开 `🧩 目录-xxx.md`，获取当前文件列表
2. **读取文件内容**：逐个读取目录中引用的文件，理解每个文件的实际内容
3. **选择分类维度**：根据文件内容特点，选择最合适的分类维度
4. **分类文件**：将文件分配到各个分类中
5. **重写目录文件**：按照格式规则重写，确保编码为 UTF-8

## 格式规则（必须严格遵守）

1. 第一行：`# 🧩 目录-xxx`（文件标题）
2. YAML frontmatter（title, type: directory, tags）
3. `## 📌 一句话总结` — 用一句话说明目录内容
4. 分类：带 emoji 的二级标题，如 `## 📁 基础流程`、`## 🏠 小区信息`
5. 文件：`- [[文件名]] — 描述`（描述可选，简洁说明文件用途）
6. 分类之间：空一行
7. 每个分类下至少 1 个文件，空分类不写
8. 末尾：`[[🏠 home-xxx]]` 链接回主页
9. **不要修改文件名**，保持原样

### 标准模板

```markdown
# 🧩 目录-xxx

---
title: 🧩 目录-xxx
type: directory
tags: [目录]
---

## 📌 一句话总结
{一句话说明这个目录包含什么内容}

## 📁 {分类1名称}
- [[文件1]] — 简短说明
- [[文件2]]

## 📁 {分类2名称}
- [[文件3]] — 简短说明

## 📁 其他
- [[文件4]]

[[🏠 home-xxx]]
```

## 分类优先级

1. **流程阶段**（项目/案件/学习/开发/产品有明确流程时）
2. **时间轴**（有时间顺序时）
3. **重要程度**（通用场景）
4. **内容类型/属性**（辅助分类）

### 分类维度参考

根据目录内容特点选择：

| 目录类型 | 推荐分类维度 |
|----------|--------------|
| 项目类（成立业委会） | 流程阶段：前期准备 → 进行中 → 完成 |
| 工具类（AI工具） | 内容类型：教程 → 参考 → 经验 |
| 知识类（AI学习） | 知识领域：基础 → 进阶 → 实战 |
| 个人类（leo） | 属性：账号 → 工作 → 阅读 → 成长 |

## 文件排序

- 系统文件、基础材料 → 前
- 参考资料、其他 → 后
- 同分类内按重要程度排序，同等按时间（最新在前）

## 注意事项

- 编码必须为 UTF-8
- 不修改任何被引用文件的内容
- 分类名称要简洁直观，符合"用户找文件时的第一反应"
- 如果某个文件难以归类，放入"其他"分类

## 操作记录

目录更新操作记录到 `⚓新增文件记录.md`，格式如下：

```markdown
### YYYY年M月D日
- 【优化】🧩 目录-xxx.md D:\Obsidian\LeoDiary\🧩 目录-xxx.md（重新分类）
```

追加写入，不覆盖已有记录。

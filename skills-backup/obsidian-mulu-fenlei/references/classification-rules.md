# 目录分类规则参考

## 格式规则

1. 第一行：文件标题，用 `#` 开头
2. 分类：直接写文字，不要加 `#`、`-`、`---` 等任何符号
3. 文件：用 tab 缩进 + `[[文件名]]` 链接
4. 分类之间：空一行
5. 每个分类下至少有1个文件，空分类不要写

## 分类维度

### 1. 按重要程度（推荐优先使用）

| 分类名称 | 说明 | 示例 |
|---------|------|------|
| 系统文件 | 系统配置、入口文件 | AGENTS、AI指令、README |
| 基础材料 | 核心内容、必须掌握 | 合同、配置文件、核心文档 |
| 核心功能 | 主要功能、常用操作 | 主要功能、核心流程 |
| 工具配置 | 工具安装、配置教程 | 软件配置、环境搭建 |
| 参考资料 | 补充说明、延伸阅读 | 案例、参考资料、研究分析 |
| 其他 | 不属于以上分类 | 杂项 |

### 2. 按时间轴（适合有时间顺序的内容）

| 分类名称 | 说明 | 示例 |
|---------|------|------|
| 早期/初始 | 最早的内容 | 项目启动、初始配置 |
| 中期/发展 | 中间阶段 | 项目执行、问题处理 |
| 后期/完成 | 最近的内容 | 项目收尾、最新版本 |
| 最新 | 最新的内容 | 最新更新、最新版本 |

### 3. 按流程阶段（适合有明确流程的内容）

| 内容类型 | 分类顺序 |
|---------|---------|
| 项目类 | 启动 → 执行 → 收尾 → 复盘 |
| 案件类 | 基础材料 → 一审 → 二审 → 再审 |
| 学习类 | 入门 → 进阶 → 高级 → 精通 |
| 开发类 | 需求 → 设计 → 开发 → 测试 → 部署 |
| 产品类 | 调研 → 设计 → 开发 → 上线 → 运营 |

### 4. 按内容类型（通用分类）

| 分类名称 | 说明 | 适用场景 |
|---------|------|---------|
| 教程 | 操作步骤、使用指南 | 软件使用、工具配置 |
| 配置 | 环境配置、参数设置 | 开发环境、软件配置 |
| 问题解决 | 常见问题、解决方案 | Bug修复、问题排查 |
| 技巧 | 使用技巧、最佳实践 | 效率提升、经验总结 |
| 对比 | 方案对比、工具对比 | 技术选型、工具选择 |
| 笔记 | 学习笔记、读书笔记 | 知识记录、学习总结 |
| 模板 | 模板文件、参考模板 | 代码模板、文档模板 |
| 案例 | 实际案例、参考案例 | 项目案例、学习案例 |

### 5. 按使用频率（适合常用内容）

| 分类名称 | 说明 |
|---------|------|
| 常用 | 每天/每周使用 |
| 偶尔 | 每月使用几次 |
| 罕见 | 很少使用，但需要时找不到 |

### 6. 按内容属性（适合技术内容）

| 分类名称 | 说明 | 适用场景 |
|---------|------|---------|
| 前端 | 前端相关 | 前端开发、UI设计 |
| 后端 | 后端相关 | 后端开发、API设计 |
| 数据库 | 数据库相关 | 数据库设计、SQL优化 |
| 部署 | 部署相关 | 服务器、CI/CD |
| 测试 | 测试相关 | 测试用例、自动化测试 |

### 7. 按账号类型（适合账号管理）

| 分类名称 | 说明 | 示例 |
|---------|------|------|
| 主流 | 常用账号 | Google、微软、苹果 |
| 开发 | 开发相关账号 | GitHub、GitLab |
| AI | AI相关账号 | ChatGPT、Claude |
| 网盘 | 网盘账号 | Dropbox、OneDrive |
| 应用 | 应用账号 | 笔记、密码管理 |
| 浏览器 | 浏览器账号 | Chrome、Edge |
| 其他 | 其他账号 | 不常用账号 |

### 8. 按工具类型（适合工具管理）

| 分类名称 | 说明 | 示例 |
|---------|------|------|
| 效率工具 | 提升效率 | AutoHotkey、PowerToys |
| 开发工具 | 开发辅助 | VSCode、IDE |
| 系统工具 | 系统管理 | 系统优化、清理 |
| 网络工具 | 网络相关 | 代理、VPN |
| 设计工具 | 设计相关 | 设计软件、图片处理 |

## 分类优先级

1. 首先判断内容是否有明确的流程阶段（按流程阶段分类）
2. 其次判断内容是否有时间顺序（按时间轴分类）
3. 然后判断内容的重要性（按重要程度分类）
4. 最后按内容类型或属性分类

## 文件排序规则

1. 每个分类内，按重要程度排序
2. 同等重要的，按时间排序（最新的在前）
3. 不确定的，按文件名排序
4. 系统文件、基础材料放前面
5. 参考资料、其他放后面

## 常见错误

1. ❌ 分类前加 `#`、`-`、`---` 等符号
2. ❌ 文件前加 `-` 符号
3. ❌ 空分类没有文件
4. ❌ 分类之间没有空行
5. ❌ 文件没有用 `[[]]` 链接格式
6. ❌ 文件没有用 tab 缩进
